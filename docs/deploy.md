# Deployment

## First production target: Render Blueprint

The repository now includes a root `Dockerfile` and `render.yaml` so the MVP can
deploy as one Docker web service with managed Postgres, managed Redis, and a
persistent disk for badge media.

The deployed URL shape is:

- Issuer app: `https://easy-credential.onrender.com/app`
- API: `https://easy-credential.onrender.com`
- Public credentials: `https://easy-credential.onrender.com/c/{slug}`
- Uploaded/generated badge media: `https://easy-credential.onrender.com/media/...`

If the Render service name or custom domain changes, update these environment
values in `render.yaml` or the Render dashboard:

```bash
APP_URL=https://your-domain.example/app
API_URL=https://your-domain.example
PUBLIC_URL=https://your-domain.example
CORS_ORIGINS=https://your-domain.example
```

## Required production secrets

Render will prompt for `sync: false` values during Blueprint setup, or you can
add them in the service environment after creation.

```bash
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...
STRIPE_SECRET_KEY=...
STRIPE_PUBLISHABLE_KEY=...
STRIPE_WEBHOOK_SECRET=...
OPENAI_API_KEY=...
```

`SECRET_KEY` is generated by Render in the Blueprint. Do not rotate it casually:
existing sessions and OAuth state cookies depend on it.

## Google OAuth values

For the default Render service name:

- Authorized JavaScript origin: `https://easy-credential.onrender.com`
- Authorized redirect URI: `https://easy-credential.onrender.com/auth/google/callback`

If using a custom domain, replace the host exactly.

## Stripe webhook

Point Stripe to:

```text
https://your-domain.example/stripe/webhook
```

Then set `STRIPE_WEBHOOK_SECRET` from the endpoint signing secret.
The wallet UI uses Stripe Payment Element, so both `STRIPE_SECRET_KEY` and
`STRIPE_PUBLISHABLE_KEY` must be set for production recharge. If either is
missing in production, `POST /orgs/{id}/wallet/recharge` returns
`503 stripe_not_configured`.

## Notes

- The Docker image builds the React app with `VITE_APP_BASE=/app` so dashboard
  routes do not collide with API routes such as `GET /credentials`.
- `RUN_MIGRATIONS=true` runs `alembic upgrade head` on container start.
- `DATABASE_URL` values that start with `postgres://` or `postgresql://` are
  normalized to SQLAlchemy's `postgresql+psycopg://` driver URL.
- Badge media is stored on the mounted disk at `/app/media`. For a later
  multi-instance deployment, move badge asset writes to S3/R2 and set
  `CDN_BASE_URL` to the public CDN host.
